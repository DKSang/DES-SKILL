# Phase 04 Done Gate Checklist — Data Product Definition

Use this checklist before marking Phase 04 Data Product Definition as Done.

## Required Inputs

- [ ] Data Product Specification exists (`_des-output/planning-artifacts/04-data-product-specification.md`).
- [ ] Phase 04 support plan exists (`_des-output/phase-support-plans/phase-04-support-plan.md`).
- [ ] Phase 04 evidence pack exists (`_des-output/evidence/phase-04/phase-04-evidence-pack.md`) or evidence is waived with reason.
- [ ] Artifact revision report exists (`_des-output/implementation-artifacts/phase-04-artifact-revision.md`).
- [ ] Handoff to Phase 05 exists (`_des-output/phase-handoffs/phase-04-to-05-handoff.md`).
- [ ] Workflow status file is available.
- [ ] Upstream Phase 03 handoff is reviewed.

## Artifact Completion

- [ ] Data product name and intent are defined.
- [ ] Primary consumer and supported use case are defined.
- [ ] Product outputs and first release output are clear.
- [ ] Product boundary and non-scope are documented.
- [ ] Business question and requirement mapping is documented.
- [ ] Service, freshness, trust, and access expectations are documented.
- [ ] Owner, steward, and lifecycle status are defined or explicitly marked Draft/Risk.
- [ ] Data contract expectations are captured at high level only.
- [ ] Governance and security considerations are captured.
- [ ] Source needs for Phase 05 are explicit enough to assess candidate sources.
- [ ] Assumptions, risks, and open questions are explicit.
- [ ] The specification does not design schemas, tables, ingestion pipelines, or code.

## Support Work and Evidence Validation

- [ ] All required support work listed in customize.toml is complete or waived with reason:
  - Phase 03 handoff review
  - Product boundary check
  - Consumer and use case alignment check
  - Product output check
  - Service expectation check
  - Trust expectation check
  - Ownership and stewardship check
  - Lifecycle status check
  - Phase 05 source need readiness check
  - Phase 04 done gate check
  - Phase 04 to 05 handoff
- [ ] Evidence pack validates the critical decisions (E-001 to E-009) or accepted risk is explicitly recorded.
- [ ] Data Product Specification is revised from evidence.

## Risk and Open Question Handling

- [ ] Risks are documented.
- [ ] Accepted risks have owner or explicit acceptance.
- [ ] Open questions are documented.
- [ ] Carry-forward risks are included in handoff.

## Workflow Status

- [ ] Workflow status update is prepared.
- [ ] Workflow status v2 sections are updated.

## Gate Result

Choose one:

- [ ] Pass
- [ ] Pass with risks
- [ ] Fail
- [ ] Blocked

## Completion

- [ ] If Pass: Phase 04 may be marked Done.
- [ ] If Pass with risks: Phase 04 may be marked Done only if risks are carried forward in the handoff.
- [ ] If Fail: required fixes are listed.
- [ ] If Blocked: blocker owner and next action are listed.
