## Evidence Quality

- [ ] Missing evidence is marked Unknown.
- [ ] Design readiness is separated from production readiness.
- [ ] Business value is not claimed without evidence.
- [ ] Adoption is not claimed without stakeholder or usage evidence.
- [ ] Quality readiness is not claimed without quality/test results.
- [ ] Production readiness is not claimed without release, security, and operational evidence.
- [ ] Known risks are visible.
- [ ] Blocking risks have owner/action.
- [ ] Next iteration priorities are specific.

## Guardrails

- [ ] The artifact does not implement fixes.
- [ ] The artifact does not rewrite upstream artifacts directly.
- [ ] The artifact does not claim success based only on documentation completeness.
- [ ] The artifact does not hide blockers.
- [ ] The artifact does not recommend production release with unresolved P1 quality/security/contract/CI/CD blockers.
- [ ] Unknown is used instead of false confidence.
