## Evidence Integrity

- [ ] Missing evidence is marked `Unknown`, `Missing`, or `Design only`, not `Green`.
- [ ] Production readiness is not claimed without release/test/security evidence.
- [ ] Business value is not claimed without business evidence or stakeholder evidence.
- [ ] Adoption is not claimed without usage or acceptance evidence.
- [ ] Quality readiness is not claimed without quality evidence or clear design-only limitation.
- [ ] Serious risks are visible.
- [ ] Open questions are visible.
- [ ] Accepted risks have owner/review expectation where needed.

---

## Done Gate Decision

Choose one:

- [ ] Pass
- [ ] Pass with risks
- [ ] Fail
- [ ] Blocked

Pass only if:

- [ ] Project Evaluation Report exists.
- [ ] Evidence availability summary exists.
- [ ] Readiness scorecard exists.
- [ ] Final decision is supported by evidence.
- [ ] Risks and limitations are documented.
- [ ] Next iteration or closeout path is documented.
- [ ] Evidence pack exists or evidence is waived with reason.
- [ ] Final closeout exists.
- [ ] Workflow status is updated.

Pass with risks only if:

- [ ] Missing or weak evidence is documented.
- [ ] Risks are accepted explicitly.
- [ ] Production readiness is not claimed without evidence.
- [ ] Final closeout status is realistic.

Fail or Blocked if:

- [ ] Evaluation scope is missing.
- [ ] Original goals are not referenced.
- [ ] Success criteria are missing.
- [ ] Readiness decision is unsupported.
- [ ] Missing evidence is treated as success.
- [ ] Serious risks are hidden.
- [ ] Next iteration recommendation is missing.
- [ ] Final closeout decision is missing.
- [ ] Artifact claims production readiness without release/test/security evidence.
- [ ] Final closeout is missing.
