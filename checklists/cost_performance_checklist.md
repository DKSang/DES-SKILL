# Cost And Performance Checklist

- [ ] Bottlenecks are measured before optimization.
- [ ] Partitioning and clustering match access patterns.
- [ ] File sizes and compression are appropriate.
- [ ] Incremental strategy avoids unnecessary full refreshes.
- [ ] Expensive queries have plans reviewed.
- [ ] Materialized views or caches have refresh ownership.
- [ ] Compute sizing and concurrency are reviewed.
- [ ] Cost monitors and alert thresholds are defined.
