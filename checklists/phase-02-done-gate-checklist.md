# Phase 02 Done Gate Checklist — Business Questions

Use this checklist before marking Phase 02 Business Questions as Done.

## Required Inputs

- [ ] Business Question Catalog exists (`_des-output/planning-artifacts/02-business-question-catalog.md`).
- [ ] Phase 02 support plan exists (`_des-output/phase-support-plans/phase-02-support-plan.md`).
- [ ] Phase 02 evidence pack exists (`_des-output/evidence/phase-02/phase-02-evidence-pack.md`) or evidence is waived with reason.
- [ ] Artifact revision report exists (`_des-output/implementation-artifacts/phase-02-artifact-revision.md`).
- [ ] Handoff to Phase 03 exists (`_des-output/phase-handoffs/phase-02-to-03-handoff.md`).
- [ ] Workflow status file is available.
- [ ] Upstream Phase 01 handoff is reviewed.

## Artifact Completion

- [ ] Each P1 question maps to a consumer.
- [ ] Each P1 question maps to a target decision or use case.
- [ ] Each P1 question has a priority assigned.
- [ ] Each P1 question is inside approved scope or explicitly marked as risk, open, or deferred.
- [ ] Each P1 question has expected answer format.
- [ ] Candidate metrics are marked as candidates, not final KPI formulas.
- [ ] Freshness expectations and grain hints are captured where relevant.
- [ ] Source hints are marked as hints, not final source decisions.
- [ ] Out-of-scope questions are separated from the main list.
- [ ] Assumptions, risks, and open questions are explicit.
- [ ] The catalog does not define final KPI formulas, source choices, table designs, or code.

## Support Work and Evidence Validation

- [ ] All required support work listed in customize.toml is complete or waived with reason:
  - Phase 01 handoff review
  - Question quality review
  - Consumer mapping check
  - Decision mapping check
  - Scope alignment check
  - Priority validation
  - Answerability risk check
  - Phase 02 done gate check
  - Phase 02 to 03 handoff
- [ ] Evidence pack validates the critical decisions (E-001 to E-006) or accepted risk is explicitly recorded.
- [ ] Business Question Catalog is revised from evidence.

## Risk and Open Question Handling

- [ ] Risks are documented.
- [ ] Accepted risks have owner or explicit acceptance.
- [ ] Open questions are documented.
- [ ] Carry-forward risks are included in handoff.

## Workflow Status

- [ ] Workflow status update is prepared.
- [ ] Workflow status v2 sections are updated.

## Gate Result

Choose one:

- [ ] Pass
- [ ] Pass with risks
- [ ] Fail
- [ ] Blocked

## Completion

- [ ] If Pass: Phase 02 may be marked Done.
- [ ] If Pass with risks: Phase 02 may be marked Done only if risks are carried forward in the handoff.
- [ ] If Fail: required fixes are listed.
- [ ] If Blocked: blocker owner and next action are listed.
