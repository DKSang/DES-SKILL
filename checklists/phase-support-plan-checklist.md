# Phase Support Plan Checklist

Use this checklist to verify that a phase support plan is ready before running support skills.

## Required Context

- [ ] Phase number and phase name are identified.
- [ ] Phase core skill is identified.
- [ ] Initial phase artifact path is listed.
- [ ] Upstream handoff is listed or marked not applicable.
- [ ] Workflow status file is referenced.
- [ ] Current phase status is recorded.

## Upstream Inputs

- [ ] Required upstream artifacts are listed.
- [ ] Missing upstream inputs are documented.
- [ ] Partial upstream inputs are documented.
- [ ] The plan does not silently assume missing upstream decisions.

## Initial Artifact Review

- [ ] Confirmed items are summarized.
- [ ] Assumptions are listed.
- [ ] Open questions are listed.
- [ ] Known risks are listed.
- [ ] Items requiring validation are identified.

## Validation Planning

- [ ] Each validation item has an ID.
- [ ] Each validation item explains why it matters.
- [ ] Criticality is assigned.
- [ ] Required evidence is defined.
- [ ] Critical validation items are not deferred without accepted risk.

## Support Skill Selection

- [ ] Required support skills are listed.
- [ ] Optional support skills are listed if useful.
- [ ] Waived support skills include reason.
- [ ] Deferred support skills include accepted risk.
- [ ] Support skill execution order is defined.
- [ ] Dependency between support skills is documented.

## Evidence Planning

- [ ] Evidence output directory is defined.
- [ ] Expected evidence files are listed.
- [ ] Evidence producer is identified.
- [ ] Evidence-to-validation mapping is clear.

## HALT Conditions

- [ ] HALT conditions are listed.
- [ ] Missing access is treated as a blocker when critical.
- [ ] Contradictory evidence requires artifact revision.
- [ ] Unvalidated downstream assumptions are not allowed.

## Completion

- [ ] Next support action is recommended.
- [ ] The phase is not marked Done at this stage.
- [ ] The support plan is ready for execution.
