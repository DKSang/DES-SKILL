# Phase 01 Done Gate Checklist — Business Discovery

Use this checklist before marking Phase 01 Business Discovery as Done.

## Required Inputs

- [ ] Business Discovery Brief exists (`_des-output/planning-artifacts/01-business-discovery-brief.md`).
- [ ] Phase 01 support plan exists (`_des-output/phase-support-plans/phase-01-support-plan.md`).
- [ ] Phase 01 evidence pack exists (`_des-output/evidence/phase-01/phase-01-evidence-pack.md`) or evidence is waived with reason.
- [ ] Artifact revision report exists (`_des-output/implementation-artifacts/phase-01-artifact-revision.md`).
- [ ] Handoff to Phase 02 exists (`_des-output/phase-handoffs/phase-01-to-02-handoff.md`).
- [ ] Workflow status file is available.

## Artifact Completion

- [ ] Project intent is clear or marked as open question.
- [ ] Primary consumer is clear or marked as open question.
- [ ] Target decision or use case is clear or marked as open question.
- [ ] Expected value is documented.
- [ ] MVP scope and non-scope are documented.
- [ ] Initial success criteria are documented.
- [ ] Assumptions, risks, and open questions are explicit.
- [ ] The Brief does not design schemas, pipelines, architecture, contracts, dashboards, ML models, or code.

## Support Work and Evidence Validation

- [ ] All required support work listed in customize.toml is complete or waived with reason:
  - Context source review
  - Business intent validation
  - Stakeholder consumer check
  - Scope boundary check
  - Success criteria check
  - Phase 01 done gate check
  - Phase 01 to 02 handoff
- [ ] Evidence pack validates the critical decisions (E-001 to E-005) or accepted risk is explicitly recorded.
- [ ] Business Discovery Brief is revised from evidence.

## Risk and Open Question Handling

- [ ] Risks are documented.
- [ ] Accepted risks have owner or explicit acceptance.
- [ ] Open questions are documented.
- [ ] Carry-forward risks are included in handoff.

## Workflow Status

- [ ] Workflow status update is prepared.
- [ ] Workflow status v2 sections are updated.

## Gate Result

Choose one:

- [ ] Pass
- [ ] Pass with risks
- [ ] Fail
- [ ] Blocked

## Completion

- [ ] If Pass: Phase 01 may be marked Done.
- [ ] If Pass with risks: Phase 01 may be marked Done only if risks are carried forward in the handoff.
- [ ] If Fail: required fixes are listed.
- [ ] If Blocked: blocker owner and next action are listed.
