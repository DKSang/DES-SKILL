# Phase Evidence Pack Checklist

Use this checklist to verify that a phase evidence pack is sufficient for artifact revision and Done Gate review.

## Metadata

- [ ] Phase number and phase name are identified.
- [ ] Phase core skill is identified.
- [ ] Initial artifact is referenced.
- [ ] Support plan is referenced.
- [ ] Evidence directory is listed.
- [ ] Evidence status is recorded.

## Evidence Inventory

- [ ] Each evidence item has an ID.
- [ ] Each evidence item has a name.
- [ ] Each evidence item has a producer.
- [ ] Each evidence item has a type.
- [ ] Each evidence item has a status.
- [ ] Each evidence item maps to validation item or decision.

## Validation Coverage

- [ ] Each critical validation item has evidence.
- [ ] Missing evidence is documented.
- [ ] Partial evidence is documented.
- [ ] Waived evidence has accepted risk.
- [ ] Evidence coverage is clear enough for Done Gate review.

## Technical Evidence

- [ ] Technical probes or spikes include method or command.
- [ ] Inputs and outputs are recorded.
- [ ] Results are marked Pass, Fail, Partial, or Blocked.
- [ ] Limitations are documented.
- [ ] Impact on phase artifact is documented.

## Data Evidence

- [ ] Sample data source is referenced.
- [ ] Schema information is captured when relevant.
- [ ] Data quality snapshot is captured when relevant.
- [ ] Freshness, nulls, duplicates, outliers, or consistency are checked when relevant.
- [ ] Limitations are documented.

## Research Evidence

- [ ] Source or document is referenced.
- [ ] Key findings are summarized.
- [ ] Licensing or terms are documented when relevant.
- [ ] Impact on phase artifact is documented.

## Review Evidence

- [ ] Reviewer or review agent is identified.
- [ ] Reviewed item is listed.
- [ ] Findings have severity.
- [ ] Required actions are listed.
- [ ] Critical findings are not ignored.

## Decision Evidence

- [ ] Decisions based on evidence are recorded.
- [ ] Rationale is documented.
- [ ] Reversibility is documented.
- [ ] Draft or open decisions are clearly marked.

## Gaps and Risks

- [ ] Evidence gaps are listed.
- [ ] Risks found from evidence are listed.
- [ ] Carry-forward risks are marked.
- [ ] Missing evidence that blocks the phase is clearly identified.

## Completion

- [ ] Evidence pack result is selected.
- [ ] Next action is recommended.
- [ ] The artifact revision requirements are listed.
