# Checklist — Epic Catalog

## Required Context

- [ ] Data Product Specification is available or Draft continuation is accepted.
- [ ] Data Source Inventory is available or Draft continuation is accepted.
- [ ] Architecture Decision Record is available or Draft continuation is accepted.
- [ ] Ingestion Specification is available or Draft continuation is accepted.
- [ ] Gold Layer Specification is available or Draft continuation is accepted.
- [ ] Transformation Specification is available or Draft continuation is accepted.
- [ ] Data Quality Specification is available or Draft continuation is accepted.
- [ ] CI/CD and Testing Specification is available or Draft continuation is accepted.
- [ ] P1/P2 implementation scope is clear or marked Draft.
- [ ] Existing workflow status is considered where available.

## Epic Coverage

- [ ] Epic creation scope is defined.
- [ ] Epic design principles are defined.
- [ ] Input artifact coverage is documented.
- [ ] Implementation themes are extracted.
- [ ] Epic inventory is created.
- [ ] Every epic has an ID.
- [ ] Every epic has a name.
- [ ] Every epic has a goal.
- [ ] Every epic has business/technical value.
- [ ] Every epic maps to DES artifacts.
- [ ] Every epic has included scope.
- [ ] Every epic has excluded scope.
- [ ] Every epic has candidate story areas.
- [ ] Every epic has dependencies.
- [ ] Every epic has acceptance criteria.
- [ ] Every epic has risk/blocker section.
- [ ] Every epic has priority.
- [ ] Every epic has release group.
- [ ] Every epic has status.

## Delivery Coverage

- [ ] Foundation epic exists.
- [ ] Source ingestion epic exists.
- [ ] Bronze epic exists.
- [ ] Silver epic exists.
- [ ] Gold epic exists.
- [ ] Contract and quality epic exists.
- [ ] Orchestration and observability epic exists.
- [ ] Semantic and serving epic exists where relevant.
- [ ] Lineage/metadata/governance/security epic exists.
- [ ] CI/CD/testing/release epic exists.
- [ ] Cost/performance epic exists or is marked not applicable/deferred.

## Traceability and Dependency

- [ ] Epic-to-DES artifact mapping is documented.
- [ ] Epic dependency map is documented.
- [ ] P1 epics are distinguishable from P2/later epics.
- [ ] Contracted outputs are represented.
- [ ] Quality gates are represented.
- [ ] Serving outputs are represented.
- [ ] Governance/security constraints are represented.
- [ ] Release/testing concerns are represented.

## Guardrails

- [ ] Epic catalog does not contain detailed implementation code.
- [ ] Epic catalog does not replace story catalog.
- [ ] Epic catalog does not contain sprint plan.
- [ ] Epics are not untraceable guesses.
- [ ] Missing context is marked Draft/Risk/Blocked.
- [ ] Epics are neither too small as tasks nor too broad as entire project.
- [ ] Story creation is deferred to `des-create-story`.

## HALT Validation

- [ ] Upstream artifact completeness HALT resolved or catalog remains Draft.
- [ ] Epic creation scope HALT resolved or catalog remains Draft.
- [ ] Implementation boundary HALT resolved or catalog remains Draft.
- [ ] P1/P2 priority HALT resolved or priorities remain Draft.
- [ ] Epic granularity HALT resolved or catalog remains Draft.
- [ ] Missing critical context HALT resolved or affected epic remains Draft/Blocked.
- [ ] Story-level request HALT resolved if applicable.

## Result

Choose one:

- [ ] Passed
- [ ] Passed with risks
- [ ] Blocked
