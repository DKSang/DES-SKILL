# Checklist — Implementation Plan

## Required Context

- [ ] Dev Task Breakdown exists or Draft continuation is accepted.
- [ ] Selected stories/tasks are identified.
- [ ] Task dependencies are available or marked Draft.
- [ ] Test tasks are available or marked Draft/Risk.
- [ ] Quality/contract/security tasks are available or marked N/A/Risk.
- [ ] Sprint/story context is considered where available.

## Implementation Plan Coverage

- [ ] Implementation Plan Summary is included.
- [ ] Implementation Scope is defined.
- [ ] Selected Stories and Tasks are listed.
- [ ] Execution Strategy is defined.
- [ ] Dependency Sequence is documented.
- [ ] Parallelization Opportunities are documented.
- [ ] Implementation Phases are defined.
- [ ] Checkpoint Plan is included.
- [ ] Validation and Test Execution Order is defined.
- [ ] Quality/Contract/Security Gate Timing is defined.
- [ ] Documentation/Metadata/Status Update Plan is included.
- [ ] Rollback/Abort/Escalation Conditions are defined.
- [ ] Coding Agent Handoff Prompt is included.
- [ ] Implementation Readiness Decision is included.
- [ ] Next Support Skill is recommended.

## Safety and Execution Quality

- [ ] Blocked tasks are not scheduled as normal executable work.
- [ ] Task order respects dependencies.
- [ ] Validation occurs before Done/release.
- [ ] Quality/contract/security gates are not skipped.
- [ ] Documentation/status updates are included.
- [ ] Parallelization does not violate dependencies.
- [ ] Stop/abort conditions are explicit.
- [ ] Handoff prompt includes constraints and references.
- [ ] Implementation readiness is evidence-based.

## Guardrails

- [ ] The artifact does not write implementation code.
- [ ] The artifact does not claim implementation completion.
- [ ] The artifact does not perform code review.
- [ ] The artifact does not create new stories or sprint scope.
- [ ] Exact file paths are not invented without repo context.
- [ ] Missing context is marked Draft/Risk/Blocked.
- [ ] Code review is recommended only after implementation exists.

## HALT Validation

- [ ] Dev task breakdown availability HALT resolved or plan remains Draft.
- [ ] Implementation scope HALT resolved or plan remains Draft.
- [ ] Task dependency sequence HALT resolved or plan remains Draft.
- [ ] Parallelization HALT resolved or plan remains conservative.
- [ ] Validation/test order HALT resolved or plan remains Draft/Risk.
- [ ] Quality/security gate timing HALT resolved or plan remains Draft/Risk.
- [ ] Repo context HALT resolved or file guidance remains logical only.
- [ ] Code implementation request HALT resolved if applicable.

## Result

Choose one:

- [ ] Passed
- [ ] Passed with risks
- [ ] Blocked
