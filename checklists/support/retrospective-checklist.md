# Checklist — Retrospective

## Required Context

- [ ] Retrospective scope is defined.
- [ ] Sprint Plan is considered or missing evidence is documented.
- [ ] Story Catalog is considered or missing evidence is documented.
- [ ] Code Review Report is considered where relevant.
- [ ] Release Readiness Report is considered where relevant.
- [ ] Workflow status is considered where available.
- [ ] Missing evidence is marked Unknown.

## Retrospective Coverage

- [ ] Retrospective Summary is included.
- [ ] Retrospective Scope is included.
- [ ] Planned vs Completed Review is included.
- [ ] Story Outcome Review is included.
- [ ] Release Readiness Outcome is included or marked N/A/Unknown.
- [ ] What Went Well is documented.
- [ ] What Did Not Go Well is documented.
- [ ] Blockers and Root Causes are documented.
- [ ] Artifact Quality Feedback is documented.
- [ ] Support Skill Workflow Feedback is documented.
- [ ] Technical Lessons Learned is documented.
- [ ] Data Quality/Contract/Governance Lessons are documented.
- [ ] CI/CD/Release Lessons are documented.
- [ ] Stakeholder/User Feedback is documented or marked Unknown.
- [ ] Action Items are documented.
- [ ] Next Sprint Recommendations are documented.
- [ ] Correct Course Recommendations are documented.
- [ ] Workflow Status Update Recommendations are documented.
- [ ] Next Support Skill is recommended.

## Decision Safety

- [ ] Completed work is not claimed without evidence.
- [ ] Release success is not claimed without release evidence.
- [ ] Missing stakeholder feedback is marked Unknown.
- [ ] Blockers include root cause or investigation action.
- [ ] Action items have owner or owner placeholder.
- [ ] Next iteration direction is explicit.
- [ ] Correct-course route is recommended when upstream issue caused blocker.

## Guardrails

- [ ] The report does not implement fixes.
- [ ] The report does not rewrite upstream artifacts directly.
- [ ] The report does not create new stories directly.
- [ ] The report does not perform code review.
- [ ] The report does not deploy/release.
- [ ] The report does not blame individuals.
- [ ] Unknown evidence is not treated as success.

## HALT Validation

- [ ] Retrospective scope HALT resolved or report remains Draft.
- [ ] Planned/completed evidence HALT resolved or evidence gaps are marked.
- [ ] Release outcome evidence HALT resolved or outcome is Unknown/N/A.
- [ ] Blocker root cause HALT resolved or investigation action exists.
- [ ] Stakeholder feedback HALT resolved or marked Unknown/action item.
- [ ] Next iteration direction HALT resolved or report remains Draft.
- [ ] Fix request HALT resolved if applicable.

## Result

Choose one:

- [ ] Passed
- [ ] Passed with risks
- [ ] Blocked
