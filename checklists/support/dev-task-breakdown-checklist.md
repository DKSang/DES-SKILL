# Checklist — Dev Task Breakdown

## Required Context

- [ ] Story Catalog exists or Draft continuation is accepted.
- [ ] Story Readiness Report exists or Draft continuation is accepted.
- [ ] Selected story is marked Ready for Task Breakdown or risk is explicitly accepted.
- [ ] Story acceptance criteria are available.
- [ ] Story test expectations are available.
- [ ] Relevant DES artifacts are checked.
- [ ] Repo/folder context is checked if file-level guidance is requested.

## Task Breakdown Coverage

- [ ] Dev Task Breakdown Summary is included.
- [ ] Task Breakdown Scope is defined.
- [ ] Selected Story Context is included.
- [ ] Implementation Context is included.
- [ ] Affected Areas are documented.
- [ ] Ordered Task List is included.
- [ ] Task Details are included.
- [ ] Test Task List is included.
- [ ] Quality/Contract/Governance Tasks are included or marked N/A.
- [ ] Dependency and Sequencing Notes are included.
- [ ] Task Acceptance Checks are mapped to story acceptance criteria.
- [ ] Implementation Risks are documented.
- [ ] Coding Agent Handoff Prompt is included.
- [ ] Ready for Implementation Decision is included.

## Task Quality

- [ ] Every task has a task ID.
- [ ] Every task has a name.
- [ ] Every task has a type.
- [ ] Every task has a purpose.
- [ ] Every task has an expected output.
- [ ] Every task references relevant artifacts.
- [ ] Every task has validation/check.
- [ ] Every task has dependencies or explicitly says none.
- [ ] Blocked tasks are not marked Ready.
- [ ] Tasks do not exceed story scope.
- [ ] Test tasks are not omitted.
- [ ] Documentation/metadata tasks are included where relevant.
- [ ] Quality/contract/security tasks are included where relevant.

## Guardrails

- [ ] The artifact does not write production code.
- [ ] The artifact does not create new stories unnecessarily.
- [ ] The artifact does not create sprint plan.
- [ ] The artifact does not perform code review.
- [ ] Exact file paths are not invented without repo context.
- [ ] Quality/security/contract work is not removed to simplify implementation.
- [ ] Blockers are visible.

## HALT Validation

- [ ] Ready story availability HALT resolved or breakdown remains Draft.
- [ ] Story selection HALT resolved or breakdown remains Draft.
- [ ] Task breakdown scope HALT resolved or breakdown remains Draft.
- [ ] Repository structure HALT resolved or file guidance remains logical only.
- [ ] Implementation boundary HALT resolved or tasks remain Draft.
- [ ] Test task expectation HALT resolved or breakdown remains Draft/Risk.
- [ ] Quality/contract/security task HALT resolved or breakdown remains Draft/Risk.
- [ ] Code implementation request HALT resolved if applicable.

## Result

Choose one:

- [ ] Passed
- [ ] Passed with risks
- [ ] Blocked
