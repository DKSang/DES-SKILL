# Checklist — Sprint Planning

## Required Context

- [ ] Epic Catalog exists or Draft continuation is accepted.
- [ ] Story Catalog exists or Draft continuation is accepted.
- [ ] Workflow status is considered where available.
- [ ] Sprint scope is defined.
- [ ] Sprint goal is defined.
- [ ] Sprint duration/capacity handling is defined.
- [ ] Candidate stories are identified.

## Sprint Plan Coverage

- [ ] Sprint Plan Summary is included.
- [ ] Sprint Planning Scope is included.
- [ ] Sprint Goal is included.
- [ ] Planning Inputs are documented.
- [ ] Sprint Capacity and Assumptions are documented.
- [ ] Candidate Story Pool is documented.
- [ ] Story Selection Strategy is documented.
- [ ] Selected Sprint Stories are documented.
- [ ] Stretch Stories are documented or marked N/A.
- [ ] Deferred Stories are documented.
- [ ] Blocked Stories are documented.
- [ ] Dependency Order is documented.
- [ ] Sprint Backlog is documented.
- [ ] Readiness Check Plan is documented.
- [ ] Sprint Acceptance Criteria are documented.
- [ ] Definition of Ready is included.
- [ ] Definition of Done is included.
- [ ] Risks and Mitigations are documented.
- [ ] Execution Guidance is included.
- [ ] Next Support Skill is recommended.

## Story Selection Quality

- [ ] Selected stories support the sprint goal.
- [ ] Selected stories respect dependency order.
- [ ] P1 or dependency-unblocking stories are prioritized.
- [ ] Blocked stories are not committed without risk handling.
- [ ] Stretch stories are separated from committed stories.
- [ ] Sprint scope is realistic for stated capacity.
- [ ] Testing, quality, governance, or CI/CD work is not ignored.
- [ ] Story readiness check is planned before task breakdown.
- [ ] Deferred stories have reasons.
- [ ] Blocked stories have required actions.

## Guardrails

- [ ] Sprint plan does not create new stories unnecessarily.
- [ ] Sprint plan does not include coding task breakdown.
- [ ] Sprint plan does not include implementation code.
- [ ] Sprint plan does not perform code review.
- [ ] Sprint plan does not approve release.
- [ ] Sprint plan does not ignore Story Catalog statuses.
- [ ] Sprint plan does not ignore blockers.
- [ ] Sprint plan does not overload capacity.
- [ ] Sprint plan does not mix unrelated goals without reason.

## HALT Validation

- [ ] Epic Catalog availability HALT resolved or plan remains Draft.
- [ ] Story Catalog availability HALT resolved or plan remains Draft.
- [ ] Sprint scope HALT resolved or plan remains Draft.
- [ ] Sprint goal HALT resolved or plan remains Draft.
- [ ] Sprint duration/capacity HALT resolved or plan remains Draft.
- [ ] Story selection strategy HALT resolved or plan remains Draft.
- [ ] Blocked story handling HALT resolved or blocked stories remain outside commitment.
- [ ] Dependency conflict HALT resolved or plan remains Draft.
- [ ] Readiness requirement HALT resolved or selected stories remain pending readiness.
- [ ] Task breakdown request HALT resolved if applicable.

## Result

Choose one:

- [ ] Passed
- [ ] Passed with risks
- [ ] Blocked
