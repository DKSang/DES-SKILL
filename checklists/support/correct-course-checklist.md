# Checklist — Correct Course

## Required Context

- [ ] Issue source is identified or user-provided.
- [ ] Issue symptom is documented.
- [ ] Severity is assigned or marked Unknown.
- [ ] Relevant support artifacts are considered where available.
- [ ] Relevant DES artifacts are considered where needed.
- [ ] Existing workflow status is considered where available.

## Correct Course Coverage

- [ ] Correct Course Summary is included.
- [ ] Issue Scope is included.
- [ ] Current Workflow State is documented.
- [ ] Symptom Register is included.
- [ ] Root Cause Analysis is included.
- [ ] Impacted Artifacts are listed.
- [ ] Impacted Epics/Stories/Sprint Items are listed.
- [ ] Correction Route is selected.
- [ ] Required Artifact Updates are listed.
- [ ] Required Support Skill Reruns are listed.
- [ ] Implementation and Release Impact is documented.
- [ ] Risk of Not Correcting is documented.
- [ ] Correction Action Plan is included.
- [ ] Workflow Status Update Recommendation is included.
- [ ] Next Support Skill is recommended.

## Decision Safety

- [ ] Blocker issues are not treated as minor.
- [ ] Root cause is not skipped.
- [ ] Impacted artifacts are not ignored.
- [ ] Release impact is considered if release is in scope.
- [ ] Accepted risk has owner/condition.
- [ ] Required actions are specific.
- [ ] Correction is not claimed complete before updates happen.
- [ ] Not all issues are blindly routed to implementation.

## Guardrails

- [ ] The plan does not directly implement fixes.
- [ ] The plan does not deploy/release.
- [ ] The plan does not rewrite all artifacts unnecessarily.
- [ ] The plan does not create new sprint scope directly.
- [ ] The plan does not hide missing evidence.
- [ ] The plan does not continue with unsafe assumptions.

## HALT Validation

- [ ] Issue symptom HALT resolved or plan remains Draft.
- [ ] Issue severity HALT resolved or marked Unknown.
- [ ] Root cause HALT resolved or investigation action exists.
- [ ] Impacted artifact HALT resolved or investigation action exists.
- [ ] Correction route HALT resolved or plan remains Draft.
- [ ] Release impact HALT resolved if release is in scope.
- [ ] Risk acceptance HALT resolved if risk is accepted.
- [ ] Fix request HALT resolved if applicable.

## Result

Choose one:

- [ ] Passed
- [ ] Passed with risks
- [ ] Blocked
