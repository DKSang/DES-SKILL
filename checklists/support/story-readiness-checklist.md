# Checklist — Story Readiness Check

## Required Context

- [ ] Story Catalog exists or Draft continuation is accepted.
- [ ] Selected stories are identified.
- [ ] Readiness scope is defined.
- [ ] Relevant DES artifacts are checked.
- [ ] Existing readiness report is checked if updating.

## Readiness Evaluation Coverage

- [ ] Story Readiness Summary is included.
- [ ] Readiness Check Scope is included.
- [ ] Readiness Criteria are included.
- [ ] Input Artifact Coverage is documented.
- [ ] Selected Stories are listed.
- [ ] Story-by-story readiness evaluation is completed.
- [ ] Missing Context Register is included.
- [ ] Dependency and Blocker Register is included.
- [ ] Quality/Contract/Governance Review is included.
- [ ] Test Readiness Review is included.
- [ ] Story Size and Clarity Review is included.
- [ ] Recommended Story Status is included.
- [ ] Required Refinements are included.
- [ ] Ready Story Handoff is included.
- [ ] Next Support Skill is recommended.

## Ready Story Requirements

For every story marked Ready for Task Breakdown:

- [ ] Story has valid story ID.
- [ ] Story maps to an epic.
- [ ] Story maps to DES artifacts.
- [ ] Goal and benefit are clear.
- [ ] Included scope is clear.
- [ ] Excluded scope is clear.
- [ ] Acceptance criteria are clear.
- [ ] Test expectations are clear.
- [ ] Dependencies are known and resolved or accepted.
- [ ] Quality expectations are defined or N/A.
- [ ] Contract expectations are defined or N/A.
- [ ] Governance/security expectations are defined or N/A.
- [ ] Story size is reasonable.
- [ ] No unresolved P1 blocker remains.

## Guardrails

- [ ] The report does not create new stories unnecessarily.
- [ ] The report does not create sprint plan.
- [ ] The report does not create task breakdown.
- [ ] The report does not write implementation code.
- [ ] Blocked stories are not recommended for task breakdown.
- [ ] Missing evidence is marked Unknown/Risk, not Ready.
- [ ] Security/quality/contract gaps are not ignored.

## HALT Validation

- [ ] Story Catalog availability HALT resolved or report remains Draft.
- [ ] Story selection HALT resolved or report remains Draft.
- [ ] Readiness scope HALT resolved or report remains Draft.
- [ ] Missing acceptance criteria HALT resolved or story not marked Ready.
- [ ] Missing test expectations HALT resolved or story not marked Ready.
- [ ] Missing quality/contract context HALT resolved or story not marked Ready.
- [ ] Missing governance/security context HALT resolved or story not marked Ready.
- [ ] Dependency blocker HALT resolved or story not marked Ready.
- [ ] Story size problem HALT resolved or story not marked Ready.
- [ ] Route back decision HALT resolved where needed.

## Result

Choose one:

- [ ] Passed
- [ ] Passed with risks
- [ ] Blocked
