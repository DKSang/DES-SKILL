# Phase Done Gate Checklist

Use this checklist before marking any DES phase as Done.

## Required Inputs

- [ ] Phase artifact exists.
- [ ] Phase support plan exists.
- [ ] Evidence pack exists or evidence waivers are documented.
- [ ] Artifact revision report exists.
- [ ] Handoff file exists or handoff draft exists.
- [ ] Workflow status file is available.
- [ ] Upstream handoff was reviewed or marked not applicable.

## Artifact Completion

- [ ] Initial artifact was created.
- [ ] Artifact was revised using evidence.
- [ ] Artifact clearly separates facts, assumptions, risks, and open questions.
- [ ] Artifact contains explicit decisions.
- [ ] Artifact does not treat unknowns as facts.
- [ ] Artifact is internally consistent.

## Support Work Completion

- [ ] Required support skills were identified.
- [ ] Required support skills were executed or waived with reason.
- [ ] Optional support skills are marked appropriately.
- [ ] Deferred support skills include accepted risk.
- [ ] Blocked support skills are recorded as blockers.

## Evidence Completion

- [ ] Critical decisions have evidence.
- [ ] Critical assumptions are validated or marked as risk.
- [ ] Evidence is stored or clearly referenced.
- [ ] Evidence coverage is full or accepted risk is documented.
- [ ] Evidence contradictions were resolved or marked as blockers.

## Risk and Open Question Handling

- [ ] Risks are documented.
- [ ] Accepted risks have owner or explicit acceptance.
- [ ] Open questions are documented.
- [ ] Blockers are documented.
- [ ] Carry-forward risks are included in handoff.

## Handoff Readiness

- [ ] Handoff to next phase exists.
- [ ] Handoff lists final decisions.
- [ ] Handoff lists approved inputs for next phase.
- [ ] Handoff lists constraints for next phase.
- [ ] Handoff lists risks carried forward.
- [ ] Handoff lists open questions.
- [ ] Handoff includes a do-not-assume list.
- [ ] Handoff recommends the next skill.

## Workflow Status

- [ ] Workflow status update is prepared.
- [ ] Phase status is not marked Done unless gate passes.
- [ ] Next recommended skill is recorded.
- [ ] Open blockers are recorded.
- [ ] Open risks are recorded.

## Gate Result

Choose one:

- [ ] Pass
- [ ] Pass with risks
- [ ] Fail
- [ ] Blocked

## Completion

- [ ] If Pass: phase may be marked Done.
- [ ] If Pass with risks: phase may be marked Done only if risks are carried forward.
- [ ] If Fail: required fixes are listed.
- [ ] If Blocked: blocker owner and next action are listed.
