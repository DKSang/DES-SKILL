# Checklist — des-learning-path

## Learning Context

- [ ] Learner goal is identified.
- [ ] Current project context is considered if available.
- [ ] Existing artifact status is considered if available.
- [ ] Generic mode is clearly marked if no artifacts exist.

## Coverage

- [ ] All 22 main DES phases are included or intentionally scoped.
- [ ] Support skills are included where relevant.
- [ ] Learning modules are clearly defined.
- [ ] Phase-to-concept mapping is included.
- [ ] Undercurrents are included.

## Learning Quality

- [ ] Each module has learning objectives.
- [ ] Each module has artifact exercises.
- [ ] Each module has quiz checkpoints.
- [ ] Each module has expected outputs.
- [ ] The path teaches through artifacts, not theory only.
- [ ] Weak areas are identified where evidence exists.

## Practicality

- [ ] The path has a recommended sequence.
- [ ] The path has next actions.
- [ ] The path supports project implementation.
- [ ] The path supports portfolio/report defense when relevant.

## Guardrails

- [ ] No phase is marked complete without evidence.
- [ ] Missing artifacts are marked Missing or Not Started.
- [ ] No long source passages are reproduced.
- [ ] The next recommended skill is included.
