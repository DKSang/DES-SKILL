# Checklist — des-learning-status-update

## Required Files

- [ ] `SOUL.md` is available.
- [ ] `knowledge/FUNDAMENTALS-MAP.md` is available.
- [ ] Learning output directory is inspected.
- [ ] Existing learning status is inspected if available.

## Evidence Scan

- [ ] Main DES artifacts are scanned.
- [ ] Learning notes are scanned.
- [ ] Artifact quizzes are scanned.
- [ ] Learning gap reports are scanned.
- [ ] Socratic coaching sessions are scanned.
- [ ] Workflow status is considered if available.

## Status Update

- [ ] All 22 phases are listed.
- [ ] Learning status is assigned per phase.
- [ ] Quiz status is assigned per phase.
- [ ] Gap status is assigned per phase.
- [ ] Coaching status is assigned per phase.
- [ ] Readiness status is assigned per phase.
- [ ] Unknown status is used when evidence is missing.

## Guardrails

- [ ] No phase is marked Ready without evidence.
- [ ] No gap is marked Resolved without evidence.
- [ ] Quiz Generated is not confused with Quiz Answered.
- [ ] Artifact Done is not confused with Learning Ready.
- [ ] Blocking gaps prevent readiness.

## Output

- [ ] `des-learning-status.md` is created or updated.
- [ ] Open learning blockers are listed.
- [ ] Weak concepts are listed.
- [ ] Recommended next learning action is included.
- [ ] Next recommended skill is included.
