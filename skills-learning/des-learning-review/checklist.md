# Checklist — des-learning-review

## Required Evidence

- [ ] `SOUL.md` is available.
- [ ] `knowledge/FUNDAMENTALS-MAP.md` is available.
- [ ] Learning status file is available or generic review mode is explicitly chosen.
- [ ] Learning outputs are inspected where relevant.

## Progress Review

- [ ] Overall progress is summarized.
- [ ] Module progress is summarized.
- [ ] Phase readiness is summarized.
- [ ] Completed learning steps are identified.
- [ ] Missing learning steps are identified.

## Strengths and Weaknesses

- [ ] Strengths are identified.
- [ ] Weak concepts are identified.
- [ ] Open gaps are identified.
- [ ] High and Blocking gaps are highlighted.
- [ ] Learning blockers are listed.

## Recommendations

- [ ] Review plan is included.
- [ ] Next learning action is included.
- [ ] Next learning skill is included.
- [ ] Next DES lifecycle action is included where safe.
- [ ] Recommendations are evidence-based.

## Guardrails

- [ ] Readiness is not claimed without evidence.
- [ ] Artifact completion is not confused with learning readiness.
- [ ] Open High or Blocking gaps are not ignored.
- [ ] Feedback is constructive.
