# Checklist — des-explain-artifact

## Source Artifact

- [ ] Source artifact path is identified.
- [ ] Source artifact exists.
- [ ] DES phase is identified.
- [ ] Artifact status is noted if available.
- [ ] Missing or weak sections are explicitly identified.

## Concept Mapping

- [ ] Related Data Engineering lifecycle concept is identified.
- [ ] Related undercurrents are identified where relevant.
- [ ] Mapping is based on `knowledge/FUNDAMENTALS-MAP.md`.
- [ ] Explanation does not invent unsupported concepts.

## Explanation Quality

- [ ] Artifact purpose is explained.
- [ ] Fundamental concept is explained.
- [ ] Business importance is explained.
- [ ] Technical importance is explained.
- [ ] Key decisions are explained.
- [ ] Trade-offs are explained.
- [ ] Upstream dependencies are explained.
- [ ] Downstream impact is explained.
- [ ] Common mistakes are explained.
- [ ] A practical example is included.

## Learning Quality

- [ ] Explanation is beginner-friendly.
- [ ] Explanation is detailed enough to support learning.
- [ ] Jargon is explained.
- [ ] Reflection questions are included.
- [ ] Next learning action is recommended.

## Safety and Source Use

- [ ] No long passages from source material are reproduced.
- [ ] Source material is used as conceptual foundation only.
- [ ] The agent does not claim the user has mastered the concept.
- [ ] The source artifact is not modified unless explicitly requested.

## Output

- [ ] Learning notes file is created or updated.
- [ ] Output follows the configured template.
- [ ] Status is marked correctly.
- [ ] Next recommended skill is included.
