# Checklist — des-gap-teacher

## Evidence

- [ ] User evidence is identified.
- [ ] Source artifact or learning notes are identified where available.
- [ ] DES phase is identified.
- [ ] The diagnosis is based on evidence, not assumptions.

## Concept Mapping

- [ ] Related lifecycle concepts are identified.
- [ ] Related undercurrents are identified where relevant.
- [ ] Mapping is based on `knowledge/FUNDAMENTALS-MAP.md`.

## Gap Diagnosis

- [ ] Correct understanding is acknowledged.
- [ ] Missing concepts are identified.
- [ ] Incorrect assumptions are identified where present.
- [ ] Weak reasoning is identified where present.
- [ ] Gap category is assigned.
- [ ] Severity is assigned.
- [ ] Evidence for each gap is included.

## Impact Analysis

- [ ] Artifact impact is explained.
- [ ] Downstream risk is explained.
- [ ] Business impact is explained where relevant.
- [ ] Technical impact is explained where relevant.

## Teaching Feedback

- [ ] Feedback is constructive.
- [ ] Study actions are recommended.
- [ ] Artifact corrections are recommended where relevant.
- [ ] Next learning action is recommended.
- [ ] No mastery claim is made without evidence.

## Guardrails

- [ ] No unsupported gaps are invented.
- [ ] Source artifact is not modified unless explicitly requested.
- [ ] No long source passages are reproduced.
