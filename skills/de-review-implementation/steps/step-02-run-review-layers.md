# Step 02 - Run Review Layers

Review through separate lenses.

- Artifact compliance: implementation matches contracts, DQ rules, transformations, lineage, governance, and architecture.
- Blind bug hunt: look for incorrect logic, boundary errors, broken assumptions, and regressions without trusting the plan.
- Edge-case hunt: empty data, late data, schema drift, retries, backfills, duplicate events, and partial failures.
- Operations review: observability, CI/CD, rollback, security, cost, and performance.
- Test review: acceptance criteria have evidence and missing tests are named.

Record candidate findings with severity and supporting evidence.
