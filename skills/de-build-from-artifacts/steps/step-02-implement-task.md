# Step 02 - Implement Task

Execute tasks in plan order.

For each task:

1. Re-read the task acceptance criteria.
2. Modify only the files needed for that task.
3. Keep data contracts, DQ rules, lineage, security, and observability requirements aligned with source artifacts.
4. Record implementation notes, decisions, and deviations immediately.
5. Stop before starting the next task if the current task cannot meet acceptance criteria.

HALT when the implementation requires new scope, new architecture, or changed requirements.
