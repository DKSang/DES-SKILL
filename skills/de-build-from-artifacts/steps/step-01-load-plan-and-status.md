# Step 01 - Load Plan And Status

Confirm the work is authorized and scoped.

1. Read `implementation-plan.md`, `implementation-story.md`, and source artifacts.
2. Check current implementation status and prior implementation log.
3. Identify planned tasks, accepted files/modules, validation commands, non-goals, and allowed sections.
4. Inspect working tree before editing.
5. Mark Status as In Progress in `implementation-story.md` and the implementation log.

Allowed sections in `implementation-story.md`:

- Status.
- Tasks / Subtasks checkboxes.
- Dev Agent Record.
- File List.
- Review Findings status.
- Change Log.

HALT if there is no approved plan, no `implementation-story.md`, or if existing changes make the planned scope ambiguous.
