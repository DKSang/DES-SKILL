# Step 01 - Load Plan And Status

Confirm the work is authorized and scoped.

1. Read `implementation-plan.md` and source artifacts.
2. Check current implementation status and prior implementation log.
3. Identify planned tasks, accepted files/modules, validation commands, and non-goals.
4. Inspect working tree before editing.
5. Mark status as in-progress in the implementation log or status artifact.

HALT if there is no approved plan or if existing changes make the planned scope ambiguous.
