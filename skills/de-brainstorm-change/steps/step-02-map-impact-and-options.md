# Step 02 - Map Impact And Options

Convert the request into explicit choices.

1. Classify the change: feature, scope adjustment, bug/incident follow-up, optimization, governance/security, release blocker, or artifact cleanup.
2. Map affected artifacts, including contracts, transformation designs, DQ rules, orchestration, lineage, CI/CD, and release readiness.
3. Identify affected consumers, owners, operational SLAs/SLOs, and rollback implications.
4. Draft at least two options when there is meaningful uncertainty.
5. Record tradeoffs in the decision log: value, risk, complexity, tests, and artifact updates.

HALT if options would break an accepted contract or governance constraint without explicit user approval.
