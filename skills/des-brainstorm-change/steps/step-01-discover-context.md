# Step 01 - Discover Context

Build the factual base before proposing a change.

1. Read the user request and restate the change in one sentence.
2. Read workflow status when available.
3. Read relevant planning artifacts, implementation artifacts, and prior review or verification reports.
4. Identify the current lifecycle point: planning, implementation, review, verification, release, or post-release.
5. Record known facts, assumptions, and missing context separately.

HALT if the request requires code changes but no approved planning artifact or change authority exists.
