# Step 01 - Gather Context

Build the review packet.

1. Identify changed files from git status/diff or implementation log.
2. Read implementation plan, implementation log, and relevant DES artifacts.
3. Read tests, validation output, and evidence if present.
4. Identify acceptance criteria and artifact compliance obligations.
5. Note missing context before reviewing.

Do not start review from the diff alone; DES review needs artifact context.
