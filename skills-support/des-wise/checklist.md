# Checklist — des-wise

## Intent

- [ ] User intent is identified.
- [ ] Intent confidence is stated when useful.
- [ ] Skill family is selected.
- [ ] Ambiguity is handled safely.

## Context

- [ ] Workflow status is checked if available.
- [ ] Learning status is checked if relevant and available.
- [ ] Existing artifacts are considered where relevant.
- [ ] Missing evidence is clearly stated.

## Routing

- [ ] One primary skill is recommended.
- [ ] The reason for the recommendation is explained.
- [ ] Required inputs are listed.
- [ ] Expected output is listed.
- [ ] Alternative skills are listed only when helpful.
- [ ] The route does not skip required phases.

## Safety

- [ ] The agent does not invent completed artifacts.
- [ ] The agent does not invent learning readiness.
- [ ] The agent does not route to implementation without design evidence.
- [ ] The agent does not route to release without review evidence.
- [ ] The agent recommends status update when status is unknown.

## Output

- [ ] Copy-paste prompt is included.
- [ ] Next recommended skill is clearly stated.
- [ ] Status update is recommended when needed.
